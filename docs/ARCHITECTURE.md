# Web3 EIP Playground - 整体架构文档

## 一、项目定位

**Web3 EIP Playground** 是一个通过游戏学习 ERC/EIP 标准的 Web3 教育平台。

核心理念：**Learn by Doing, Not Reading**

用户价值：

- 玩互动小游戏理解 EIP-1559、EIP-7702 等标准
- 阅读像素风漫画，降低学习门槛
- 向 AI 导师提问，获得通俗解答
- 通关后获得 NFT 成就徽章

## 四、前端架构

### 设计风格：Retro-Futuristic Arcade

- **视觉**：8-bit/16-bit 像素风 + 霓虹灯效果
- **配色**：深空蓝紫背景 + 霓虹黄/青/品红高亮
- **交互**：游戏化体验，按钮像街机按键
- **字体**：Press Start 2P (标题) + 等宽字体 (正文)

### 页面结构

```
┌─────────────────────────────────────────────────────────────────────┐
│                              路由                                   │
├─────────────────────────────────────────────────────────────────────┤
│  /              →  Home.tsx        首页 (Hero + 功能网格)           │
│  /eip/:id       →  EIPDetail.tsx   EIP 详情页 (漫画 + 游戏 + 内容)   │
│  /404           →  NotFound.tsx    404 页面                         │
└─────────────────────────────────────────────────────────────────────┘
```

## 十一、安全要点

| 要点 | 说明 |
|------|------|
| **Minter 私钥** | 仅存储在后端环境变量中，不暴露给前端 |
| **Gas 费用** | 由项目方 Minter 钱包承担，用户无需付费 |
| **防重放** | 双重保障：后端幂等检查 + 合约 `claimUsed[claimId]` |
| **随机性** | 弱随机（block.prevrandao），适合展示类场景 |
| **通关验证** | 后端应验证用户确实完成了游戏，防止伪造请求 |
| **用户体验** | 用户全程无需签名交易，体验流畅 |
